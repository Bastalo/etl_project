# Парсер вакансий hh.ru
Скрипт для автоматизированного сбора вакансий с платформы **hh.ru**, сохранения данных в базу **PostgreSQL** и отправки ежедневных отчетов в **Telegram**.

## Основные возможности

* **Инкрементальная загрузка:** Использует механизм "High Water Mark" (максимальную дату из БД), чтобы исключить дубликаты.
* **Повторы при ошибках:** Благодаря библиотеке `tenacity`, скрипт устойчив к временным сбоям сети или блокировкам.
* **Гибкая настройка:** Конфигурация вынесена в YAML-файлы через фреймворк `Hydra`.
* **Уведомления:** В конце работы бот присылает отчет в Telegram с количеством найденных вакансий или деталями ошибки.
* **Логирование:** Скрипт записывает логи, которые помогают отслеживать процесс сбора данных и диагностировать ошибки.

## Предварительные требования

1.  Установленный **PostgreSQL**.
2.  Созданная схема `etl` в вашей базе данных.
3.  Созданный Telegram-бот (через [@BotFather](https://t.me/botfather)) и полученные `token` и `chat_id`.

## ⚙️Настройка

Ожидаемая структура папок:
```text
project_root/
│
├── conf/
│   └── config.yaml
└── main.py  # ваш скрипт
```

**Пример содержания `conf/config.yaml`:**
```yaml
hh_api:
  text: "МТС"
  area: 1
  search_field: "company_name"

database:
  url: "postgresql+psycopg2://etl:password@localhost:5432/your_db"
  table: "etl.hh_search"

logging:
  level: "INFO"
  file: "tmp.text"

telegram:
  token: "123456789:ABCDEF..."
  chat_id: "987654321"
```

## Установка

Установите все необходимые зависимости одной командой:

```bash
pip install requests beautifulsoup4 polars pendulum sqlalchemy psycopg2-binary hydra-core tenacity tqdm
```

## Запуск и автоматизация

### Ручной запуск
```bash
python main.py
```

### Настройка в Windows Task Scheduler
Для ежедневного запуска:
1.  Создайте задачу в Планировщике задач.
2.  **Триггеры:** Указать условие запуска, например, каждый день в 12:30. 🕧
3.  **Действие:** Указать запуск программы и полный путь к вашему скрипту.
4.  Проверить что папка с файлом конфигурации лежит в той же директории, что и сам скрипт.

## Структура данных в PostgreSQL
Данные сохраняются в таблицу `etl.hh_search`. Основные поля:
* `request_dttm` — Время запроса (UTC).
* `vacancy_id` — ID вакансии на HH.
* `vacancy_title` — Название вакансии.
* `company_title` — Название компании.
* `publication_time` — Дата и время публикации.
* `snippet` — Описание/требования (в формате JSON).
* `responses_count` — Количество откликов на момент сбора.
